En en archivo .env tenemos los datos de configuracion (incluyendo los de conexion a la DB)

Las migraciones son archivos que se encargan de modificar el schema de una base de datos
Pueden servirnos para llevar un registro de las modificaciones.
Laravel recomienda artisan(asistente de linea de comandos de laravel)

la idea es que la migraciones siempre puedan ser invertibles

Seeding
nos da la posibilidada de sembrar las bases de datos con data, utilizando las clases de siembra
sembrar = poner valores iniciales
generalmente son datos iniciales o de prueba
comando para crear: `php artisan make:seeder BookSeeder`
comando para ejecutar: `php artisan db:seed`
